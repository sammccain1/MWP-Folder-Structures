# templates/ — Reusable Consulting Assets

You are in the **templates directory**. This contains reusable, fully anonymized documents and frameworks that can be adapted for any client engagement.

## Subdirectory Guide

| Directory | Purpose |
|---|---|
| `Proposals/` | Proposal and SOW templates. Adapt for each new engagement before sending. |
| `Reports/` | Report and analysis templates. Includes structured data audit, summary, and recommendation formats. |
| `Frameworks/` | Analytical and delivery frameworks — methodology documents, scoring rubrics, project archetypes. |

## Rules for This Directory

- All templates must be **fully anonymized** — zero traces of specific clients, projects, or data
- Templates use `kebab-case.md`: `engagement-proposal-template.md`, `data-audit-report.md`
- When adapting a template for a client, copy it into `Client-[Name]/` — never edit the master template
- Templates should indicate clearly with `[PLACEHOLDER]` where client-specific values go
- Version templates with a changelog comment at the top if they evolve significantly
