# docs/changelog/ — Context

This directory holds semantic versioning logs.

## Purpose

Track what features, fixes, and breaking changes were made in every release.

## Format

Create a new file per version: `vX.Y.Z.md`

Within the file, adhere to the following categories:
- **Added:** New features or tools.
- **Changed:** Updates to existing functionality.
- **Fixed:** Bug fixes.
- **Removed:** Deprecated pieces of the platform.
