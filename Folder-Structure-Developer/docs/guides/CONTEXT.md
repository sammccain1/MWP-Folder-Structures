# docs/guides/ — Context

This directory stores "How To" instructions meant for human developers.

## Purpose

Guides serve as onboarding tools, specific playbooks, and troubleshooting walkthroughs. 

## Rules

- Guides must be **actionable**. Write "How to deploy X" rather than "An overview of deployment".
- Use `kebab-case.md` for filenames.
- Provide explicit bash code snippets that can be copy/pasted.
